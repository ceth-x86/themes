# Color specification

Base surfaces:

- Paper #FFFCF0 for main backgrounds
- Panel #F8F4EA for secondary surfaces
- Panel Muted #F2EDE3 for subtle containers and inactive areas
- Border #CFC9B9 for standard separators, card edges, and form outlines
- Border Subtle #E6E0D0 for decorative dividers and very soft separation only

Text hierarchy:

- Ink #100F0F for primary text and main content
- Text Secondary #403E3C for secondary text and labels
- Text Muted #6D6C67 for low-emphasis text, placeholders, comments, and subtle UI elements

Semantic accents:

- Blue #205EA6 for actions, links, active states, and interactive emphasis
- Teal #197870 for structure, types, grouped entities, and system relationships
- Magenta #A02F6F for logic, commands, keywords, and control emphasis
- Ochre #8D6400 for numbers, data, metrics, search highlights, and attention states that are not errors

Status colors:

- Green #5B7500 for success, confirmation, and positive outcomes
- Red #C13D31 for errors, destructive actions, danger, and failed states

Usage guidance:

- prioritize Ink and Text Secondary for long-form reading and dense UI copy
- use Text Muted for metadata, helper copy, and placeholders, not for critical instructions
- Blue and Magenta are safe for links and inline text accents on light surfaces
- Teal, Ochre, Green, and Red are tuned to stay readable across the light surface set, but still work best for short labels, badges, icons, and compact emphasis
- use Border for controls and structural separation; use Border Subtle only where contrast is intentionally quiet
- avoid relying on color alone for destructive, success, or warning states
- prefer tonal separation and spacing before adding more saturation

General style:

- warm, calm, editorial, typographic
- avoid neon saturation
- avoid pure white and pure black
- use accents intentionally, not decoratively
- prefer tonal separation over harsh contrast
